## Day 3 (수요일) — 원인 분석 (Authority: 전문성 확보)

### [Day 3 / YouTube 본편 스크립트]
**(길이: 약 10~12분 분량, A4 3장 기준)**

**[제목]: 이 규제 하나로 주가 30% 갈린다. 미국 반도체 수출규제가 삼성/하이닉스에게 주는 진짜 기회 (Feat. 규제 격차 분석)**
**(영상 속 보이스톤: 전문성을 갖춘, 믿음직한 선배 투자자 톤)**

**[🔥 후크 (0~30초)]**
"지금 반도체 주식에 돈을 넣고 계시다면, 잠시 멈추세요. 여러분은 지금 '기술적 상승'만 보고 있습니다. 하지만 이 시장의 진짜 엔진은 **정책(Policy)입니다.** 미국 정부가 수출 규제를 강하게 풀수록, 오히려 한국 기업들이 새로운 기회 지점, 즉 '규제 격차(Regulatory Gap)'에서 거대한 알파 수익을 얻게 됩니다. 오늘은 그 작동 원리와 당신이 당장 주목해야 할 종목 후보군 3곳을 공개합니다."

**[챕터 1: 규제 이벤트 발생 (문제 제기)]**
*   **(내용):** 미국의 수출 통제가 왜, 어떤 기술(예: 첨단 패키징 장비)에 집중하는지 설명. 이로 인해 글로벌 공급망이 어떻게 '병목 현상'을 겪게 되는지 시각화 자료와 함께 보여준다.
*   **(핵심 메시지):** 규제는 일방적 제재가 아니라, 시장의 **구조적 재편(Structural Reorganization)** 신호탄이다.

**[챕터 2: 시장 격차 발생 (분석/원인)]**
*   **(내용):** 미국이 막아내는 기술과 규제가 오히려 아시아나 유럽 등 다른 지역에 '수요 폭발'을 일으키는 과정을 분석한다. 이 Gap이 바로 한국 기업들에게 기회가 된다고 설명한다. (예: A국가에서 못하는 기술을 B국가가 대안으로 급부상)
*   **(시각 자료 강조):** 규제 전후의 글로벌 공급망 지도를 애니메이션으로 보여주며, 빨간색(규제 리스크), 초록색(기회 영역) 구분을 명확히 한다.

**[챕터 3: 수혜 종목 및 투자 시나리오 (핵심 분석)]**
*   **(내용):** 규제를 우회하거나, 혹은 규제가 만들어낸 병목 현상을 해결하는 핵심 역할을 하는 국내 기업들(삼성/하이닉스 관련 협력사 포함)을 선정하여 그 근거를 제시한다. 단순히 '좋다'가 아니라, **'왜 이 시점에서 돈이 들어오는지'** (정책적 명분과 수주 공백 메우기)에 집중해야 한다.
*   **(자료 활용):** 정책 면역 점수(Policy Immunity Score) 같은 가상의 구조화 지표를 도입하여 전문성을 높인다.

**[챕터 4: 당신의 행동 계획 (Actionable Insight)]**
*   **(내용):** 시청자가 이 영상을 보고 바로 할 수 있는 구체적인 투자 액션 플랜을 제시한다. 분산 투자의 중요성, 플레이북 활용법 등을 안내한다.
*   **(CTA):** "오늘 영상에서 다룬 복잡한 논리를 한 장의 체크리스트로 압축했습니다. **플레이북 다운로드 링크는 댓글 상단에 고정되어 있으니 지금 바로 확인하세요.**"

from typing import Dict, Any

class ContentScheduler:
    """
    최종적으로 최적화된 콘텐츠를 받아 스케줄링 및 배포 상태를 관리하는 핵심 서비스.

    """
    def __init__(self, db_path="mock_db/scheduled_content.json"):
        self.db_path = db_path
        # 초기화 시 Mock DB가 없으면 생성 (실제로는 Redis나 PostgreSQL 사용)
        try:
            with open(self.db_path, 'r') as f:
                self.schedule = json.load(f)
        except FileNotFoundError:
            print("⚠️ Mock DB 파일이 없어 초기 스케줄러를 생성합니다.")
            self.schedule = []

    def schedule_content(self, content_data: Dict[str, Any], scheduled_time: datetime) -> str:
        """콘텐츠를 특정 시간대에 예약하고 DB에 저장합니다."""
        # 필수 유효성 검사: Title, Platform, Time이 모두 있어야 함.
        if not all([content_data.get('optimizedMetadata', {}).get('title'), 
                    content_data.get('platform'), scheduled_time]):
            raise ValueError("필수 메타데이터가 누락되어 스케줄링 할 수 없습니다.")

        new_entry = {
            "id": f"SCH-{len(self.schedule) + 1}",
            "title": content_data['optimizedMetadata']['title'],
            "platform": content_data['platform'],
            "scheduledAt": scheduled_time.isoformat(),
            "status": "Scheduled",
            "content_payload": content_data # 전체 콘텐츠 로드
        }
        self.schedule.append(new_entry)
        self._save_to_mock_db()
        return new_entry['id']

    def simulate_publishing(self, entry_id: str):
        """실제 게시물 업로드 과정을 시뮬레이션하고 오류를 보고합니다."""
        for item in self.schedule:
            if item['id'] == entry_id and item['status'] != "Published":
                print(f"⏳ {item['platform']}에 [{item['title'][:20]}...] 게시물 업로드를 시뮬레이션합니다...")
                # 실제 API 호출 대신, 가상의 성공/실패 로직 구현 (예: 외부 API Rate Limit 초과)
                if "RateLimit" in item['content_payload']['raw_data']: # 오류 유도 테스트를 위한 임시 필드 검사
                    item['status'] = "Failed"
                    item['publishingDetails']['errorLog'] = "API Rate Limit Exceeded. 1시간 후 재시도 필요."
                    return False, item['publishingDetails']['errorLog']
                else:

                    print(f"✅ 성공적으로 게시 완료. (ID: {entry_id})")
                    self._save_to_mock_db()
                    return True, None

            json.dump(self.schedule, f, ensure_ascii=False, indent=4)
