# 📋 콘텐츠 자동 배포 시스템 (Content Scheduler) 구축 프로젝트 계획서

## ✨ 개요 및 목표
*   **프로젝트명:** AI Stock Factory 통합 콘텐츠 자동 배포 & 스케줄링 시스템 (CSAS)
*   **최종 목표:** 모든 에이전트(Writer, Designer, Leo 등)가 생산한 원시 콘텐츠를 표준화된 데이터 구조를 통해 취합하고, 각 플랫폼별 최적의 시점에 자동으로 예약 게시 및 보고하는 완벽히 자동화된 파이프라인을 구축한다.
*   **기간:** 2주 (총 10 영업일 기준)

---

## 📅 프로젝트 마일스톤 및 일정 (Timeline)

| 주차 | 날짜 예상 | 마일스톤 목표 | 핵심 결과물 | 주요 점검 사항 |
| :---: | :---: | :--- | :--- | :--- |
| **Week 1** | Day 1 ~ Day 5 | **[Phase 1] 기반 설계 및 코어 개발 (MVP 준비)** | 표준 데이터 스키마(JSON Schema) 확정 및 백엔드 파서 V1.0 완성. | 모든 콘텐츠의 입력 포맷을 통일할 수 있는지 검증. |
| **Week 2** | Day 6 ~ Day 10 | **[Phase 2] 통합, API 연동 및 테스트 (운영 버전)** | 플랫폼별 API 연동 완료 및 자동 스케줄링 모듈 완성. 관리 대시보드(Frontend) UI/UX 확정. | 실제 게시까지의 오류 케이스(Error Handling) 대응 시스템 구축. |
| **Week 2 말** | Day 10 | **[Phase 3] 최종 검수 및 배포 준비** | 자동화된 End-to-End 테스트 완료 및 운영 매뉴얼 작성. | 모든 팀원이 실제 환경에서 시나리오 기반 테스트를 수행해야 함. |

## 💎 필수 자원 체크리스트 (Required Resources & API Keys)

**✅ 개발 환경/기술적 자원:**
1.  [Developer] 통합 Content JSON Schema 파일 (최종 확정본).
2.  [Secretary] 전용 테스트 계정 및 관리 대시보드 접근 권한.
3.  [전체] 각 플랫폼(YT, IG 등)의 Sandbox API Key 및 인증 토큰.

**✅ 콘텐츠/디자인 자원:**
1.  [Designer] 모든 콘텐츠에 통일적으로 적용될 **최종 디자인 시스템 가이드라인 (v2.0)**.
2.  [Writer] 최소 3주치 분량의 고품질, 표준화된 원본 스크립트 데이터 세트.
